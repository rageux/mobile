Windows Phone 8.1 brings new codes and methods to serve as the basic utilities of the developers.

One of the essential must-have codes for every developer is connect to HTTP and Representational State Transfer (REST) web services. A web service is a solution given by websites owners to interact easily with the data in a way to visualize, edit, remove or store information from different platforms using a simple call of the web service.

This example is about visualizing RSS feeds:

This feature gives you mobile application dynamicity by displaying latest news from an RSS feed. It is the most common feature needed by developers to make their first apps displaying the latest news/article/data from a website.

Disclaimer: it’s highly recommended that you use a RSS feed that you own, or get a permission from the provider website. As a teacher and a developer I don’t recommend using this example in applications using RSS feeds of registered brands or teams website without a permission.

Building the Sample

This sample is packaged as a Windows Phone 8.1 project. The C# source code can be used to a Windows 8.1 project.

To create a Windows Phone 8.1 project, you must be running the Windows Phone SDK 8.1 on Visual Studio 2013 (Update 3 if possible). You can download the latest version of the SDK from http://dev.windowsphone.com/downloadsdk.

Network capabilities

This sample requires that network capabilities be set in the Package.appxmanifest file to allow the app to access the network at runtime. These capabilities can be set in the app manifest using Microsoft Visual Studio.

Build the sample

3.Follow the steps for the version of the sample you want

Description

While Webclient class is commonly used in Windows Phone Silverlight to handle the communication (sending and receiving basic requests to webservice: GET, PUT, POST, DELETE) the HttpClient class is used to send and receive these requests over HTTP.

It is the main class for sending HTTP requests and receiving HTTP responses from a resource identified by a URI. This class can be used to send a GET, PUT, POST, DELETE, and other requests to a web service. Each of these requests is sent as an asynchronous operation. The HttpClient class can be used in scenarios that use text as well as scenarios that use arbitrary streams of data.

This is the main change between Windows Phone 8.1 and the old Silverlight code used to manage RSS feeds. The Webclient is replaced by HttpClient class
